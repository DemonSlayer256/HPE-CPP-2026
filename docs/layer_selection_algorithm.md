
# Layer Selection Algorithm

## Overview
The **Layer Selection Algorithm** determines how information from lower levels of the compression pipeline is selected and propagated to higher abstraction levels during hierarchical summarization.

The compression system produces a **5-level hierarchy of summaries**, ranging from detailed section summaries to a global overview. Because information is progressively compressed, the algorithm ensures that **critical entities, relationships, and factual knowledge units are preserved across layers**.

The algorithm relies on three main signals:

1. **Atomic Knowledge Units (AKUs)** extracted during chunk processing  
2. **Entity importance scores** computed during topic extraction  
3. **Knowledge graph relationships** between entities  

These signals guide the selection of content that should appear in higher-level summaries.

---

# Hierarchical Layers

The summarization hierarchy consists of the following levels:

| Level | Description | Source |
|------|-------------|------|
| Level 4 | Section summaries | Generated from chunks |
| Level 3 | Document summaries | Aggregates section summaries |
| Level 2 | Topic summaries | Aggregates document summaries |
| Level 1 | Category summaries | Aggregates topic summaries |
| Level 0 | Global overview | Aggregates category summaries |

Each level compresses information while preserving the most important knowledge elements.

---

# Core Objectives

The layer selection algorithm ensures:

- Important entities are preserved across summarization layers
- Critical relationships between services remain intact
- Numerical values and parameters are not lost
- Summaries remain faithful to the original documents
- Information flows consistently across the hierarchy

---

# Key Inputs

The algorithm operates on structured data generated by previous modules.

## 1. Atomic Knowledge Units (AKUs)

AKUs represent minimal factual statements extracted from text.

Example:

```
(Lambda, requires, IAM Role)
(S3, triggers, Lambda)
(PutObject, returns, ETag)
```

These units provide a **ground truth layer of factual information** that should be preserved in summaries.

---

## 2. Entity Importance Scores

Each entity is assigned an importance score calculated using multiple factors:

```
Importance Score =
0.30 × TF-IDF
+ 0.25 × PageRank
+ 0.20 × Position
+ 0.15 × Specificity
+ 0.10 × Co-occurrence
```

Entities with higher scores are more likely to be included in higher-level summaries.

---

## 3. Knowledge Graph

Entities and relationships extracted from documents form a **knowledge graph**.

Example:

```
S3 ----triggers----> Lambda
Lambda ----requires----> IAM Role
S3 ----uses----> KMS Encryption
```

The graph helps identify **central services and critical relationships**.

---

# Layer Selection Strategy

At each level of summarization, the algorithm performs the following steps:

### Step 1: Aggregate Lower-Level Units
Collect all summaries and knowledge units from the previous layer.

### Step 2: Extract Candidate Entities
Extract entities from summaries, AKUs, and knowledge graph nodes.

### Step 3: Compute Importance Ranking
Rank entities using the importance scoring model.

### Step 4: Preserve Critical Relationships
Retrieve relationships between selected entities from the knowledge graph.

### Step 5: Select Supporting AKUs
Fetch factual AKUs associated with the selected entities.

### Step 6: Generate Constrained Summary
Generate the higher-level summary while enforcing constraints such as:
- Including top-ranked entities
- Preserving relationships
- Retaining numerical values and parameters

---

# Pseudocode

```python
def layer_selection(lower_layer_units):

    entities = extract_entities(lower_layer_units)

    scores = compute_importance_scores(entities)

    ranked_entities = sort_by_score(entities, scores)

    top_entities = ranked_entities[:K]

    relationships = fetch_relationships(top_entities)

    supporting_akus = fetch_akus(top_entities)

    summary = generate_summary(
        input_units=lower_layer_units,
        required_entities=top_entities,
        required_relationships=relationships,
        required_facts=supporting_akus
    )

    return summary
```
